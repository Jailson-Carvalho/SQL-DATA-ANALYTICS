
# 🎯 Filtragem Avançada de Funcionários com Operadores de Conjunto (IN e NOT IN)

Ao estruturar relatórios de Recursos Humanos ou analisar a distribuição de pessoal entre departamentos, é comum precisarmos isolar setores específicos enquanto removemos exceções ou registros irrelevantes. 

Este artigo analisa como uma consulta SQL utiliza a combinação dos operadores `IN` e `NOT IN` para extrair um grupo altamente específico de colaboradores da tabela `DimEmployee` do **AdventureWorks**.
```
```
## 💻 A Consulta SQL

O script abaixo seleciona registros de funcionários filtrando-os simultaneamente pela inclusão de departamentos e pela exclusão de nomes específicos.

```sql
SELECT *
FROM DimEmployee
--WHERE EmployeeKey IN (2,5,7)
WHERE FirstName NOT IN ('Kevin', 'Rob', 'Jossef')
  AND DepartmentName IN ('Marketing', 'Production')
ORDER BY DepartmentName;

```

## 🛠️ Análise Passo a Passo do Código

A estrutura lógica dessa query demonstra controle sobre quais dados devem entrar ou sair do resultado:

### 1. Linhas Comentadas (`--`)

```sql
--WHERE EmployeeKey IN (2,5,7)

```

O uso de dois hifens (`--`) indica um comentário em linha no SQL. O interpretador ignora esse trecho. Isso sinaliza que, originalmente, houve uma tentativa de filtrar por chaves primárias específicas (`EmployeeKey`), mas a lógica foi alterada para priorizar filtros textuais.

### 2. Exclusão de Nomes (`NOT IN`)

```sql
WHERE FirstName NOT IN ('Kevin', 'Rob', 'Jossef')

```

A cláusula `NOT IN` cria uma "lista negra". O banco de dados analisa o primeiro nome (`FirstName`) de cada colaborador; se o nome for **Kevin**, **Rob** ou **Jossef**, o funcionário é automaticamente descartado do resultado, mesmo que pertença ao departamento correto.

### 3. Inclusão de Departamentos (`AND IN`)

O operador lógico `AND` dita que a próxima condição também deve ser obrigatoriamente verdadeira:

```sql
AND DepartmentName IN ('Marketing', 'Production')

```

O operador `IN` funciona como uma "lista branca". Em vez de escrever múltiplas condições com `OR` (`DepartmentName = 'Marketing' OR DepartmentName = 'Production'`), o `IN` reduz a sintaxe agrupando os alvos. Apenas colaboradores dos setores de **Marketing** ou **Production** (Produção) avançam para o relatório.

### 4. Agrupamento Visual (`ORDER BY`)

* `ORDER BY DepartmentName`: Organiza as linhas de forma alfabética (A-Z) com base no nome do departamento. Isso agrupa todos os funcionários do Marketing primeiro, seguidos por todos os funcionários da Produção, facilitando a leitura e a exportação para ferramentas de análise.

---

## 📝 Resumo dos Conceitos Técnicos

| Comando / Operador | Tipo | Função na Query |
| --- | --- | --- |
| `--` | Comentário | Desativa trechos de código sem a necessidade de apagá-los. |
| `NOT IN` | Operador de Exclusão | Remove do resultado qualquer registro que coincida com a lista fornecida. |
| `IN` | Operador de Inclusão | Permite mapear e trazer múltiplos valores exatos de forma simplificada. |
| `AND` | Operador Lógico | Conecta filtros exigindo que a linha atenda a ambos os critérios para ser exibida. |
| `ORDER BY` | Cláusula | Classifica e organiza o resultado final (neste caso, por ordem alfabética de setor). |
